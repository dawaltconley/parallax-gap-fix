Parallax Gap Fix

This page uses Kieth Clark's Pure CSS parallax layout. It works fine without scrollbars, or with the default Mac scrollbars for trackpads, which do not change the width of the scrolling element.

Try forcing scrollbars

On Mac, you can do this by going to Settings > General and setting Show scroll bars to Always. This also happens under the default settings whenever a USB mouse is connected.

Do you see it now?

A red strip of document body peeking past the parallax background. This is because the scrollbar changes the inner width of an element, but not the total size, in which the 3D space for parallax elements is rendered.

How do I fix it?

You can try some of my fixes using the menu in the top left. The first pair modifies the perspective-origin and transform-origin properties. The second adds padding and negative margins to offset the scrollbar. Unfortunately, none of them are perfect.

Browser Inconsistencies

Safari seems to render 3D space differently than other modern browsers. Fixes 1a and 2a are slightly misaligned on Safari, while fixes 1b and 2b only work on Safari. Fix 3 removes the scrollbar entirely, but only works in webkit browsers.

Verdict?

At the moment, Fix 2a seems to be the best solution all around. Parallax elements are misaligned on Safari but still fill the 3D space so that no document body is visible. A combination of 2a and 3 would center the alignment accross all browsers at the cost of accessibility.
